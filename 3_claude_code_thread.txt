WHAT THIS WOULD GET WRONG IN PRODUCTION
================================================================

1. Batch settlements where one bank transfer covers multiple
   transactions cannot be split to individual transaction IDs
   without a line-item mapping file the bank doesn't always
   provide — causing false "missing settlement" flags.

2. UTC vs IST timezone mismatch produces false cross-month
   flags — a transaction at 11:45 PM IST on March 31 becomes
   March 30 in UTC, so UTC-based pipelines wrongly flag it
   as settled in the wrong month.

3. The duplicate check uses pandas duplicated() which loads
   all transaction IDs into memory — at 10M+ transactions
   per month this will crash or timeout and needs to be
   replaced with a SQL GROUP BY transaction_id HAVING
   COUNT(*) > 1 query on the database directly.
